# About

I enjoy roles where I can own a problem end to end: understanding requirements, designing the pipeline, implementing the solution, validating results, and improving performance.

I built batch ingestion pipelines and automated reporting workflows using **SQL, Hive, Spark, Python, and Unix shell**, often working with large, sensitive datasets across fraud, risk, customer, and merchant domains. I owned two major reports end to end and contributed improvements that reduced runtime and improved reliability.

In my thesis, I focus on extracting statistically reliable patterns from large DevOps datasets using two approaches:  
- **LLM-based pattern discovery** with structured prompting and evaluation

To make findings trustworthy and scalable, I built an automated **pattern validator tool** using a domain-specific language and **model-to-text code generation** that produces **Java validators**.

I’m actively looking for roles in **data engineering, data science, product analytics, and applied ML**, and I’m open to working across domains where data quality, automation, and measurable impact matter.
